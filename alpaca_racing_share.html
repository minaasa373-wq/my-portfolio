DX受講生むけ 開発シェア

あるぱかプリンス レーシングを
作ってみた話 4月に学んだAI・Web・GitHubが、全部つながった

コードをほとんど書けない僕でも、AIと一緒に3Dレーシングゲームが作れました。 どうやって作ったのか、使ったツールやコツを、なるべくやさしくまとめます。

01 — きっかけ

すべては4月のコースから始まった

学んだことを「使ってみたい」という気持ちが、このゲームの出発点でした。

🎓

4月「社員教育とAIコース」でWeb制作・アプリ開発を学んだ

このコースで学んだ知識が、ゲームを作るすべての土台になりました。あわせて Claude AI と GitHub も4月から使い始めました。

📸

今月のInstagram課題が終わって、暇すぎて何か作りたくなった

課題と追加投稿も終わり、せっかくなので… 栗原さんのVTuber「あるぱかプリンス」の二次創作に挑戦してみよう、と思い立ちました。

02 — 最初のアイデア

じつは最初、シューティングゲームを考えていた

ネタで作った生成画像をそのままゲームにしたかったのですが…結論から言うと断念しました。

💭

なぜ断念したか：AIエージェントでちゃんと動くものを一から作るには追加課金が必要だったり、手間がかかりそうでした。そこで別の方法を探すことに。

03 — 出発点を見つける

海外の掲示板で「使える素材」を発見

ゼロから作るのではなく、すでにあるものを土台にする。ここで GitHub が大活躍しました。

🌏

Reddit（海外の掲示板）で3Dレーシングゲームを発見

「Codex で3Dレーシングゲームを作ったよ」という投稿を見つけました。ソースコードが GitHub で公開されていたので、これを土台にしようと決めました。

🍴

「フォーク」して自分のコピーを作る

GitHubのフォーク機能で、公開されたゲームを丸ごと自分のアカウントにコピー。そこから Claude AI と一緒に、あるぱかプリンスの世界観へ作り変えていきました。

📦 元データの GitHub リポジトリ

GitHub の凄さを妻に話したら、全く伝わらなかった話（笑）

一言でいうと「コード専用のクラウド・タイムマシン」。何が便利なのか、まとめてみました。

😰
GitHub なしの世界

「最終版_本当に最後_これで決定.zip」が量産される。間違えて消したらもう戻れない。誰が何を直したかも分からない。

😎
GitHub ありの世界

変更はすべて記録され、いつでも巻き戻せる。世界中の公開コードをコピーして出発点にできる。今回もそれで作れた。

「オープンソース」だから、すごい。でも、気をつけることも。

コードを誰でも見られる・使える形で公開することをオープンソースといいます。これには良い面と注意すべき面の両方があります。

🤝
良いところ

世界中の人がコードを共有し、改良を持ち寄れる。だから開発がどんどん進む。今回のように、他の人の成果を土台にして自分の作品を作ることもできる。

🔍
気をつけるところ

誰でも公開できるぶん、未完成・動かないコードもある。なかにはウイルスが仕込まれていることもあるので、信頼できそうな相手のものか確認してから使うのが安心。

⚠️

初心者が注意する点：公開（Public）リポジトリはインターネット全体に見えてしまいます。パスワードや個人情報、APIキーなどは絶対に置かないこと。

04 — 使ったツール

素材は全部、無料のAIで用意した

コードも、音楽も、3Dモデルも、画像も。それぞれ得意なAIに任せました。

🤖Claude AI / Claude Code
ゲームのコード修正・機能追加。「〇〇したい」と日本語で伝えるだけで実装してくれる。
🎵Suno AI
BGMをAIで作曲。疾走感とサビの盛り上がりをイメージして生成。
🧊Meshy AI
画像から3Dモデル（.glb）を無料生成。キャラを車に乗せるのに使用。
🖼️画像生成AI
デフォルメキャラのシート → タイトル・リザルト画面の背景を生成。
🎵 BGMのこだわり

「うまぴょい伝説」をお手本にした疾走感ある曲

ゲーム「ウマ娘」の楽曲をAIに詳しく調べてもらってから、ネタ寄りの歌詞・疾走感・盛り上がるサビをイメージして作詞作曲。生成しながら自分でも手直しして仕上げました。

▶ BGMを聴いてみる

キャラを乗せるために作った素材

🏎️

マリオカートみたいに「車にキャラを乗せたい」と言ったら3Dモデルが必要と言われ😭 → Meshy AI で画像から3Dモデルを無料で作って解決しました。サウンドは著作権フリーの音源も活用しています。

05 — 磨き上げ

最初は40点。そこから直していった

完成度が低くても大丈夫。やり方はとてもシンプルでした。

気になるところを全部 箇条書きにする
→ Claude に渡す → まとめて直してもらう
自分でコードを一行ずつ直す必要はありません。「やりたいこと」を言葉にするだけ。
追加した演出

レースらしい盛り上げ

3-2-1-GOのスタートカウントダウン（効果音つき）、ゴール後のリザルト画面を画像つきで豪華に、記録のスクショ保存ボタンなど。

遊びやすさ改善

操作とカメラを調整

カメラをドライバー視点に近づけ、コース幅を広げてコースアウトを軽減。Escキーでいつでも操作説明が見られるように。
